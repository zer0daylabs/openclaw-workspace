# Telecom Knowledge Base

## Welcome to Your Hybrid Knowledge System

This knowledge base combines **Obsidian** (markdown-based notes) with **Graphiti** (temporal knowledge graph) to create a powerful, interconnected system for telecom and telephony domain knowledge.

### 🎯 Purpose

This vault serves as the primary knowledge repository for:
- Telecom architectures and protocols
- Telephony systems and configurations
- Network services and integrations
- Best practices and operational procedures
- Project documentation and decisions

### 📂 Structure

The vault is organized into specialized folders:

- **Core-Packages/** - Foundation software (Kamailio, FreeSWITCH, etc.)
- **Protocols/** - Communication protocols (SIP, WebRTC, H.323, etc.)
- **Features/** - Feature implementations and capabilities
- **Integrations/** - Third-party service integrations
- **Configurations/** - System configurations and setups

### 🔗 Knowledge Connections

This system uses **wikilinks** ([[term]]) to connect related concepts, creating a knowledge network that grows with your understanding. The Graphiti component adds temporal awareness, remembering when facts were established and how they evolve.

### 🛠 Tools

**Obsidian Skill** - Search, create, and manage markdown notes
- `obsidian_search "query"` - Search across all notes
- `obsidian_create "title"` - Create new notes
- `obsidian_edit "pattern" "replacement"` - Update existing notes

**Graphiti Skill** (when available) - Query temporal knowledge graph
- `graphiti_search "query"` - Search temporal facts
- `graphiti_store "fact"` - Store new facts with timestamp

### 🚀 Getting Started

1. Explore existing core package documentation
2. Use wikilinks to connect related concepts
3. Reference this file when starting new documentation
4. The agent will auto-populate knowledge as you work with telecom systems
