# Telecom Infrastructure Knowledge Base

## Core Services

### Kamailio
- **Type**: SIP Proxy Server
- **Function**: Load balancer for SIP traffic
- **Key Features**: 
  - SIP routing rules
  - Request routing and forwarding
  - Load balancing across multiple backend servers
  - SIP message manipulation

### FreeSWITCH
- **Type**: Media Server / Softswitch
- **Function**: Handles voice and video media processing
- **Key Features**:
  - Real-time voice/video streaming
  - TTS (Text-to-Speech) functionality
  - STT (Speech-to-Text) integration
  - Conference bridging
  - IVR systems

### Pipecat
- **Type**: LLM Voice Assistant Framework
- **Function**: Connects LLMs to real-time voice interactions
- **Key Features**:
  - Streaming audio processing
  - Low-latency voice AI responses
  - WebSocket-based architecture
  - Integrates with LLMs for conversational AI

### WebRTC
- **Type**: Real-time Communication Protocol
- **Function**: Browser-to-browser communication
- **Key Features**:
  - Peer-to-peer voice/video
  - No plugins required
  - NAT traversal (STUN/TURN)
  - Built-in encryption

### SIP (Session Initiation Protocol)
- **Type**: VoIP Signaling Protocol
- **Function**: Establishes and manages communication sessions
- **Key Features**:
  - Call setup/teardown
  - User location and availability
  - Call forwarding and transfer
  - Standard protocol: RFC 3261

### RTP (Real-time Transport Protocol)
- **Type**: Media Streaming Protocol
- **Function**: Transports audio/video in real-time
- **Key Features**:
  - Sequence numbering and timestamps
  - Payload type identification
  - Works with RTCP for QoS monitoring
  - Typically runs on UDP ports 10000-20000
